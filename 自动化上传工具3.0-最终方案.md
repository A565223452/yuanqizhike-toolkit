## 一、核心设计原则（不可更改）

| 原则 | 约束条款 |
|---|---|
| **UI 零改动** | `uploader.html` 布局 / 样式 / 快捷键 / 按钮文案 / id 类名 **完全不动**，只允许在原有结构上增量加 1~2 个 checkbox/toggle 级别的微型 UI |
| **API 100% 兼容** | `/api/status`、`/api/scan`、`/api/process`、`/api/commit`、`/api/icons`、`/api/test-ai`、`/api/events`(SSE) 的 URL / 请求体 / 响应体 schema **一字不改**；自定义 header `x-ai-provider` / `x-ai-key` / `x-ai-base-url` / `x-ai-model` **保留** |
| **插件化 & 组件化** | 6 阶段 Pipeline + 生命周期 Hook Bus，所有业务逻辑进 `plugins/`，单一职责、可插拔、支持独立单元测试；纯函数公共能力进 `lib/`，无状态、可单测 |
| **一键回滚兜底** | `config.flag_useLegacyPipeline = true` 随时切回旧线性代码模式，内核只做薄壳转发 |
| **密钥零泄露** | 真实 key 永不写进被 git 跟踪的文件；gateway 配置只保留 `${ENV_XXX}` 占位符；本地密钥文件一律 `.gitignore` |

## 二、3 项拍板决策

### ✅ 决策 1：autoCommit 默认关闭

| 配置项 | 默认值 | 说明 |
|---|---|---|
| `monorepo.autoCommit` | **false** | 自动同步 → 自动 AI 识别 → **只推到预览列表**，绝不在后台自动入库 |
| `monorepo.autoSyncOnStart` | true | 启动即全量扫描 monorepo diff |
| `monorepo.autoAnalyze` | true | NEW / UPGRADE 自动跑 ANALYZE 到预览 |

**入库触发唯一路径（无例外）：**
> UI 勾选卡片 → 点「确认入库」→ 二次 confirm 弹窗 → 才进入 PREFLIGHT→COMMIT→POST

**未来全自动开启白名单（需同时满足）：**
1. 静态 QC 通过率 ≥ 99.5%（近 500 次入库统计）
2. 动态深度检测累计覆盖 ≥ 50% 工具
3. 连续 30 天无功能类线上回滚

### ✅ 决策 2：gateway-main 开发模式 = tsx 直接跑

```js
// config.mjs 新增 gateway 段
gateway: {
  enabled: true,
  // 自动切换：生产 NODE_ENV=production 用 build，否则 tsx
  mode: (process.env.NODE_ENV === 'production') ? 'build' : 'tsx',
  tsxEntry:   './gateway-main/src/start-server.ts',
  buildEntry: './gateway-main/build/start-server.js',
  buildCmd:   'npm --prefix ./gateway-main run build',
  port: 8787,
  healthPath: '/health',
  startTimeoutMs: 30_000,
  maxRestarts: 3,          // 超 3 次回退直连
  restartCooldownMs: 5000,
}
```

**进程生命周期：**
- 启动：检查 node_modules → 按 mode spawn → 轮询 /health 30s → 成功 / 超时回退
- 运行中：`child.on('exit')` 按 cooldown 自动重启（<maxRestarts），每次请求前探活
- 退出：`SIGINT/SIGTERM/exit` 钩子 `SIGTERM → SIGKILL` 清理子进程

### ✅ 决策 3：密钥存储（保留前端 UI 填写习惯）

#### 3.1 四层密钥优先级

```
① 当前会话内存 Map（最高）
   └─ 刚填完未保存，进程退出即丢
② 本地加密文件 .secrets.local.json（可选）
   └─ UI 勾选 ☑️ 才写；aes-256-gcm，key = 机器 SID + %USERNAME%
③ 系统环境变量 process.env.*
   └─ 部署正式版 Docker/systemd 注入用
④ test 模式 / 占位（兜底）
```

> 🔒 **铁律（来自经验 #1155961）**
> - gateway-config.local.json 内只允许 `${ENV_XXX}` 占位符
> - 日志脱敏：只保留 key 前 4 + 后 4 位
> - `.gitignore` 永久忽略 `.secrets.local.json`、`gateway-config.local.json`

#### 3.2 UI 改动最小化

只在 AI 配置面板 apiKey 输入框下方加 1 个 checkbox：
```
☑️ 保存密钥至本地配置文件（加密存储，重启自动加载）
```

#### 3.3 鉴权做成可枚举开关（防瞎猜）

```js
// lib/auth-schemes.mjs
AUTH_SCHEMES = {
  BEARER_TOKEN:    'Bearer <apiKey>',    // 默认
  RAW_AUTH_HEADER: '<apiKey>',
  BASIC_AUTH:      'Basic <base64(u:p)>',
  QUERY_PARAM:     '?apikey=<apiKey>',
};
// 匹配逻辑：provider 前缀 + 401 响应体 error_code 客观匹配
```

### 🆕 补充 2：QC 质量检测「两层分层设计」

| 层级 | 插件名 | 触发 | 必经 | 耗 Token | 目的 |
|---|---|---|---|---|---|
| **L1 静态预检** | `qc-static-gate` | 入库前自动跑 | ✅ 是 | ❌ 0 | 挡结构/语法/资源残缺 |
| **L2 动态深度检测** | `qc-deep-browser` | 手动按钮触发 | ❌ 否 | ✅ 有 | 抓功能真的可用否 |

#### L1（必经，0 Token）：

```
必检 BLOCKER（任一 = 禁止入库）：
  · 存在 index.html
  · <title> + meta description 非空
  · 根容器 (id=app 或等价) 存在
  · 所有 .js / <script> 内嵌 JS parse 成功
  · 本地 img/link/script 引用文件存在

仅 WARNING（不阻塞）：SEO h1 / viewport / lang 等
输出：{ overallPass, blockers[], warnings[], score }
```

#### L2（手动触发，可选）：

> 触发方式：卡片右上角「🔬 深度检测」按钮 / 批量栏「🔬 已勾选批量检测」

```
流程：puppeteer-core 无头（复用系统 Chrome，不自带）
  → 加载 index.html → 2s + networkidle0
  → 采集 console.error / window.onerror
  → 关键 DOM 可见性
  → 点 1~3 个按钮冒烟
  → 收集结果喂 LLM（通过 gateway）判断严重级别 0-5

输出：{ pass, severity: CRITICAL|HIGH|MEDIUM|LOW|NONE,
        aiSummary, aiSuggestion, consoleErrors[], screenshot? }
```

L2 **不阻塞入库**，只在卡片上打颜色标签。

## 六、Plugin 编写规范（Manifest）

```js
// plugins/analyze/ai-classify.mjs
export default {
  name: 'ai-classify',
  type: 'pipeline',              // pipeline / syncer / standalone
  version: '1.0.0',
  stage: 'ANALYZE',              // SCAN/ANALYZE/PREFLIGHT/DETAIL/COMMIT/POST
  order: 'on',                   // before / on / after
  priority: 100,                 // 小值先跑
  toggle: 'analyze.aiEnabled',   // config.xxx 可关掉
  dependsOn: ['qc-static-gate?'],
  conflicts: ['legacy-ai-handler'],

  async before(ctx, items) { /* 预取 token */ },
  async on(ctx, item)     { /* 主逻辑，返回新 item */ },
  async after(ctx, items, reports) { /* SSE item-ready */ },
  async error(ctx, err, item) { /* 单失败不阻断：return item */ },
};
```

**Pipeline 6 阶段契约：**

| 阶段 | 输入 | 输出 | 失败策略 |
|---|---|---|---|
| SCAN | `{folderName, absPath}` | `ScannedItem{files,size,mtime,htmlMeta?}` | 单失败跳过 |
| ANALYZE | ScannedItem | `AnalyzedItem{category,title,desc,features,icon,zipUrl,aiOk}` | aiOk=false 继续，UI 红标 |
| PREFLIGHT | AnalyzedItem | `ValidatedItem{qcL1Pass,blockers,warnings,qcL2Result?}` | blockers 非空 → 不进后续 |
| DETAIL | ValidatedItem | `WithDetail{detailHtmlPath,detailUrl}` | 可跳过 |
| COMMIT | WithDetail | `CommittedItem{zipPath,jsonWritten,migrated,isUpgrade}` | 事务：失败自动回滚 zip |
| POST | CommittedItem[] | void | 失败只告警，不回滚 |

## 九、风险 & 回滚

| 风险 | 应对 |
|---|---|
| 插件内核引入 bug | `flag_useLegacyPipeline=true` 一键切回旧实现 |
| gateway 子进程启不来 | 超时 + maxRestarts=3 耗尽 → 静默切回直连，对用户透明 |
| 大批量 Token 爆 | 并发 3 + 安静 RPM=10 + gateway 限流 → 三层保险 |
| 密钥明文泄露 | 内存 env 注入 + aes 本地加密 + .gitignore 双保险 |
| L2 误判 | L2 永不阻塞入库，只做标签，一键清空重跑 |
| monorepo 目录结构变 | 目录映射进 config + 宽松匹配，缺失只 WARNING 不 CRASH |
