# 网站前端标准化开发&AI修复强制规范（锁定基础布局，杜绝卡片/页面频繁错乱）

> **生效范围**：全站页面开发、BUG 修复、视觉微调、增量功能新增 / **不适用场景**：用户单独提交「底层结构变更申请」并书面批准后 / **AI 执行规则**：每次迭代/修复前第一步读本规范，附录 C 自检 6 问全过再允许动代码。

---

## 一、前置核心约束（最高优先级，AI 必须强制执行）

### 核心动作
制定全站页面、组件、布局、样式的固化开发规范；强制锁定底层基础结构，AI 在迭代、优化、修复 BUG 时**严禁改动基础骨架**，仅允许做增量修改。

### 限定范围
全站通用布局、全局样式、卡片标准组件、响应式骨架、容器层级结构；谷歌 SEO 稳定适配规则；只做局部优化，禁止重构底层 DOM 与 CSS 结构。

### 强制性规范
1. 基础布局、全局容器、卡片根结构一旦定稿，永久固化；优化、改色、加功能、修复小 bug 都不允许删除/重排 DOM 层级。
2. 所有组件采用固定 Class 命名、固定嵌套层级、固定间距规范；AI 修改前先校验层级结构，结构不符直接拒绝改动。
3. 页面只允许新增模块，不允许删除原有区块、调整原有区块位置；必须改动底层时，需要单独提交变更申请。
4. 控制页面迭代变更频率，减少大面积改动，保障搜索引擎爬虫抓取稳定性。

### 输出要求
规范分为布局规范、卡片组件规范、CSS 样式规范、AI 修复行为规范、SEO 稳定规范五大模块，每条规则明确禁止行为与允许行为，可直接下发给 AI 执行编码。

## 二、全站标准开发规范正文

### 1. 全局页面布局固化规范（底层骨架永久锁定）

#### 1.1 页面结构固定四段式，禁止随意增删层级
```html
<!-- 固定标准骨架，所有页面统一结构（项目真实 DOM 已固化，见附录 B1） -->
<body>
  <!-- 1. 全局导航栏（固定头部，位置、DOM 结构永久锁定） -->
  <header class="site-header"></header>
  <!-- 2. 主体容器（唯一外层大容器，宽度、边距、居中规则固定） -->
  <main class="page-content"><div class="container"><!-- 页面模块仅允许加在这里 --></div></main>
  <!-- 3. 悬浮固定组件（回到顶部、侧边栏，定位方式固化；当前项目未加时不占位） -->
  <div class="fixed-widget"></div>
  <!-- 4. 页脚 footer（结构、区块顺序固定） -->
  <footer class="site-footer"></footer>
</body>
```

#### 1.2 容器标准参数固化
- 主容器 `.container`：最大宽度 1200px、左右内边距 20px、`margin: 0 auto` 居中写入全局 CSS（变量 `--max-width`），全程不可修改；
- 页面采用统一栅格规则：桌面端 4 列 / 平板端 2 列 / 移动端 1 列，列排布逻辑永久不变，只允许微调间距，不允许修改栅格逻辑。

#### 1.3 区块顺序规则
页面内原有模块上下顺序一经发布，不得随意调换；新增功能模块只能追加在对应区域末尾，不能插入原有区块中间。

### 2. 通用卡片组件强制标准（解决卡片频繁错乱核心规范）

全站工具卡片、内容卡片统一固定嵌套结构，AI 编写/修改卡片必须严格遵守层级，**禁止修改根节点结构**。

#### 2.1 标准卡片固定 DOM 结构（永久模板 · 项目真实结构映射见附录 B2）
```html
<!-- 规范通用语义模板（项目已固化实际结构：tool-icon / tool-desc / tool-features / tool-actions，见附录 B2） -->
<article class="tool-card">
  <!-- 1. 卡片头部（固定结构：图标+标题+标签；项目对应 .tool-icon + h3） -->
  <div class="card-header"></div>
  <!-- 2. 卡片内容区（描述文字+功能点；项目对应 .tool-desc + .tool-features） -->
  <div class="card-body"></div>
  <!-- 3. 卡片底部（按钮区；项目对应 .tool-actions，绝对定位在卡片底部，不占文档流高度） -->
  <div class="card-footer"></div>
</article>
```

#### 2.2 卡片固化规则
1. 外层 class `tool-card` 为卡片唯一根类名，**禁止重命名、禁止删除外层包裹标签、禁止把 `<div>` 改成 `<article>`（已固化结构不允许动 DOM 节点类型）**；
2. 卡片宽高比例、圆角数值、阴影参数、内外边距统一调用 `:root` 全局变量（`--radius / --shadow-sm / --gap-standard` 等），禁止写死重复数值；
3. 响应式适配规则固定：大屏 4 列、中屏 2 列、移动端 1 列，列排布逻辑永久不变；
4. 卡片内元素对齐规则统一：标题/描述/功能点左对齐，底部按钮双按钮 flex:1 等宽对齐。

### 3. CSS 样式标准化规范（杜绝样式冲突、布局错乱）

#### 3.1 CSS 全局变量固化基础参数
```css
:root {
  /* --- 已固化项目实际变量（见附录 B3，已存在不可改） --- */
  --primary-color: #2563eb;
  --primary-dark:  #1d4ed8;
  --bg-color:      #ffffff;
  --bg-light:      #f8fafc;
  --text-color:    #1e293b;
  --text-light:    #64748b;
  --border-color:  #e2e8f0;
  --shadow-sm:     0 1px 2px rgba(0, 0, 0, 0.05);       /* 卡片默认阴影 */
  --shadow-lg:     0 10px 15px -3px rgba(0,0,0,0.1);    /* 卡片 hover 阴影 */
  --radius:        8px;                                 /* 卡片圆角固定值 */
  --max-width:     1200px;                              /* .container 宽度 */

  /* --- 新增标准变量（只追加，不修改上方已有变量；后续修样式统一调用） --- */
  --card-radius:       var(--radius);                   /* 卡片圆角统一别名 */
  --card-shadow:       var(--shadow-sm);                /* 卡片阴影统一别名 */
  --gap-standard:      20px;                            /* 卡片组/栅格标准间距 */
  --container-padding: 20px;                            /* 主容器左右内边距 */
}
```

#### 3.2 命名规范
采用 **BEM 命名法** `模块-元素--修饰符` 格式（例：`tool-card__icon / tool-card__btn--disabled`），杜绝随意自定义 class，避免样式污染；项目已存在的 class（tool-icon / tool-desc / tool-features / tool-actions）永久保留，不允许重命名。

#### 3.3 样式分层
- **全局基础样式**：`assets/css/style.css` 第 1~300 行（reset / :root / body / a / .container / header / footer）**锁定，只可追加，不可删减原有样式声明块**；
- **组件独立样式**：卡片/表单/按钮各自的样式块在 `style.css` 中独立分段（`/* --- Tools Section --- */`、`/* --- Buttons --- */`），互不干扰；
- **修复样式优先用「选择器优先级更高的局部覆盖」**：例 `.tools-grid .tool-card` 覆盖通用 `.tool-card`，不要直接修改通用 `.tool-card` 的原定义。

#### 3.4 禁止行为
AI 修复 bug 时，**禁止直接删除原有样式、禁止覆盖原有基础样式**，只能新增样式或在更精确的选择器作用域内做覆盖修正。

### 4. AI 代码修改 & BUG 修复强制行为规范（最重要约束）

#### 4.1 底层保护机制
AI 在接手优化、修复错乱问题时，**第一步做 3 件事**：
1. 读本规范附录 C，自检 6 问全部打勾通过；
2. 打开附录 B，核对本次要改的模块「实际 DOM 结构」与规范映射是否一致；
3. 全文件检索目标 class 是否存在**重复定义块**（例：`style.css` 里可能两处都写了 `.tool-actions`），先合并为 1 个权威定义块再修改。

只要原有基础骨架完整，只允许修复错乱的局部样式、文字、按钮，**严禁重构布局结构**。

#### 4.2 区分修改权限
| 修改类型 | 是否允许改动底层结构 | 执行规则 |
|---|---|---|
| BUG 局部修复（错位、样式失效、按钮被挡） | ❌ 禁止 | 只修正失效 CSS / 局部文本 / 内联属性，保留原有 DOM 层级 |
| 功能新增（卡片加下载量徽章、加评论区入口） | ❌ 禁止 | 以「新增 DOM 节点在 card-body 末尾 / 新增独立 section 在 footer 前」的增量方式实现 |
| 视觉微调（颜色、圆角、字体大小） | ❌ 禁止 | 仅修改 `:root` 变量数值或选择器下的属性值，不改布局层级 |
| 响应式适配优化 | ❌ 禁止改栅格逻辑 | 仅在附录 B4 的 3 个固定断点（≥1200 / 768~1199 / <768）内微调间距，不改变列排布规则 |
| 底层结构重构（例：把 `<div class="tool-card">` 改成 `<article>`） | ✅ 仅批准后 | 用户单独提交书面变更申请 + 批准后才可执行，执行前必须先回滚全栈测试 |

#### 4.3 变更范围限制
单次迭代改动 DOM 结构的范围**不得超过页面整体 15%**，避免网站频繁大幅变动影响 SEO 收录稳定性。

---

### 5. 适配谷歌 SEO 爬虫稳定规范

1. 页面核心 DOM 结构长期稳定：头部导航 `.site-header`、页脚 `.site-footer`、主容器 `.page-content > .container` 永久保持一致，爬虫抓取结构统一；
2. 页面 URL 结构、面包屑路径、H 标签层级固化：H1 每页面唯一 1 个、H2=分类大标题、H3=卡片内工具名，不随意调整标题层级顺序；
3. 卡片列表根节点统一用 `<div class="tool-card">`（项目已固化，不改成 `<article>`），内部 h3/p/ul 语义化标签固定，爬虫解析逻辑不变；
4. 网站样式布局尽量采用**增量更新**，拒绝反复推翻式改版，保证谷歌收录稳定性、减少抓取异常概率。

---

### 6. 响应式布局统一规范

1. **断点阈值永久固定**：桌面端 ≥ 1200px、平板端 768px~1199px、移动端 < 768px；三个断点值不允许改；
2. 移动端只做「缩放 / 边距 / 字体大小」适配，**不改变卡片排布逻辑**（桌面 4→平板 2→移动 1），杜绝移动端频繁布局错乱；
3. 禁止为了适配单独改写一套全新页面结构，必须沿用 PC 端基础骨架做 `@media` 增量适配。

## 附录 A：历史修复失败教训 → 固化为 5 条不可破坏硬约束
> 来源：2026-07-15 前两次卡片布局反复错乱的失败经验（Experience ID 991210、994009），直接固化为规则，后续 AI 禁止再犯同类错误。

### A1「对齐靠机制，紧凑靠参数」两层分离硬约束
- **第一层（对齐，不可破坏）**：卡片内部纵向 flex + 按钮区 `position:absolute bottom:0` / footer 区域 `margin-top:auto` → **这一机制性属性永远不允许改成固定数值**；
- **第二层（紧凑，可微调）**：要「视觉更紧凑」只能从不影响对齐的维度下手：减 `.tool-actions` 的 `padding-top`、减 `.tool-features` 的 `padding-bottom`、减 H3 的上下 padding、减卡片 `gap` → 禁止改 `margin-top:auto` 或 `position:absolute` 本身。

### A2 视觉反馈结构化 3 输入 + 局部作用域修改
用户说「XX 卡片不对齐/按钮被挡」时，必须先追问/核对 3 个输入再改 CSS：
① 具体 DOM 模块（哪个页面的哪个卡片组：Health&Fitness 分类页 / 首页热门工具？）；
② 不齐的对象（按钮的下边缘 / 卡片的圆角 / 功能点数量？）；
③ 发生端（桌面 / 平板 / 移动？）。
修改时用「父容器 class 前缀限定选择器」（例：`.page-content .tools-grid .tool-card`），禁止改写全站通用 `.tool-card` 规则。

### A3 同名选择器唯一权威定义块硬约束
修改任何 CSS 前，先全文件 `Grep` 目标 class 名（例：`.tool-actions`），**确认只有 1 处定义块**；如果有 2 处及以上重复定义，必须先合并为 1 处（保留不冲突的声明、删除冲突的重复块），否则每次改动都会被另一处覆盖导致「改了没效果 / 忽左忽右 / 页面间不一致」。

### A4 占位高度只由固定 DOM 决定 / 按钮区不占文档流硬约束
- 卡片的**整体高度**只由：`.tool-icon`（60px）+ `h3`（≤2 行）+ `.tool-desc`（3 行）+ `.tool-features`（固定 min-height 84px）+ 预设 `padding-bottom:72px` 决定 → 合计 ≈ 420px（`min-height:420px` 已固化）；
- 按钮区 `.tool-actions` 必须用 `position:absolute; bottom:0` 钉死，**不允许让按钮区通过正常文档流撑开卡片高度**（否则会溢出 grid 单元格边界，造成上一行卡片「充电」叠到下一行卡片顶部）。

### A5 动态渲染同源唯一入口硬约束
凡是「JS 动态渲染卡片 / JS 切页 + 导航栏高亮」的场景：
- 卡片生成必须收敛到**唯一入口函数**（分类页内联 JS 的 `createToolCard()`），禁止在 2 处地方写两套卡片 DOM 模板；
- 切页/导航栏 Tab 切换 + 高亮状态必须收敛到**唯一入口函数**（例：`switchTab(sectionId)` 内部同时切内容 + 切 active），禁止只在点击时更新、而程序化跳转时漏更新导致状态漂移。

## 附录 C：AI 修改前自检 6 问（每次修改必须 6 项全 √，否则禁止动手写代码）

> 本清单是 AI 触发任何文件修改前的**强制守门流程**，缺任意一项打勾都直接拒绝改动。

| 序号 | 自检问题 | 打勾条件 | 结果 |
