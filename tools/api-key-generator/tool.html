How to Use

  1. Enter prefix (e.g. sk_, pk_, api_).
  2. Set length and choose format.
  3. Optional: separator every N chars (e.g. xxxx_xxxx).
  4. Click Generate and copy keys.

Uses crypto.getRandomValues() for secure randomness. UUID v4: 122 bits entropy (6 bits reserved). 32 alphanumeric chars = ~190 bits. 32 hex chars = 128 bits. Keys generated locally — never uploaded. Common prefixes: sk_ (secret), pk_ (publishable), live_, test_, api_. Always store API keys hashed server-side.
